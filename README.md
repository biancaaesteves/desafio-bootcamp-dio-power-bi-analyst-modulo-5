# 📊 Dashboard Gerencial para Tomada de Decisões com Power BI

## 🎯 Objetivo do Projeto

Este projeto foi desenvolvido durante o módulo **Data Analytics & Storytelling com Power BI** da Formação **Power BI Analyst (DIO)**.

O desafio teve como objetivo atualizar um relatório financeiro com foco em **experiência do usuário (UX)**, aplicando conceitos de storytelling, organização visual e navegabilidade dentro do Power BI.

O foco principal não foi apenas a criação de visuais, mas a construção de uma experiência analítica mais intuitiva, clara e estratégica para tomada de decisões.

---

# 🧠 Conceitos Aplicados

Durante o desenvolvimento do projeto, foram aplicados conceitos importantes de design analítico e Data Analytics, como:

* Storytelling com dados
* Experiência do usuário (UX)
* Organização visual de dashboards
* Segmentação de dados
* Navegabilidade entre páginas
* Estruturação visual para tomada de decisão
* Hierarquia de informação
* Proporção áurea
* Contraste e foco visual

---

# 📚 Etapas Desenvolvidas

O projeto envolveu:

### 🔹 Adequação do relatório original

Reestruturação do dashboard financeiro inicial com foco em clareza visual e experiência do usuário.

### 🔹 Ajuste de visuais

Modificação e otimização de gráficos para melhorar interpretação dos dados e leitura analítica.

### 🔹 Criação de matriz de vendas

Desenvolvimento de matriz descritiva para detalhamento das informações financeiras.

### 🔹 Estilização do relatório

Aplicação de melhorias visuais na página utilizando:

* contraste
* alinhamento
* hierarquia visual
* proporção áurea

### 🔹 Navegabilidade

Implementação de:

* botões de navegação
* menus entre páginas
* destaque visual para seleção e focalização

### 🔹 Estrutura multipágina

Construção de relatório composto por 3 páginas com navegação integrada.

---

# 🎨 Experiência do Usuário (UX)

O projeto foi desenvolvido considerando boas práticas de UX aplicadas ao Power BI.

Foram considerados fatores como:

* facilidade de leitura
* organização dos elementos
* redução de poluição visual
* destaque para informações prioritárias
* fluidez de navegação

Aqui está o ponto-chave:

> Um dashboard eficiente não é o que possui mais gráficos, mas o que permite entender os dados rapidamente.
